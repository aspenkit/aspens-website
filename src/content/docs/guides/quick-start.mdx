## Before you run it

aspens runs via `npx` — no global install needed. Requires [Node.js 20+](https://nodejs.org) and at least one backend CLI.

## Fastest setup

```bash
npx aspens doc init --recommended
```

This is the actual setup command. `--recommended` is the fastest path:

- Reuses existing target config when present
- Defaults to improving existing docs instead of prompting
- Auto-picks the generation mode based on repo size
- Installs hooks and save-tokens setup

## Full workflow

<Steps>

1. **Map the repo** (optional — just to see what aspens finds. Note, this is a rough scan, for precise mapping and domain discovery we use an LLM.)

   ```bash
   npx aspens scan
   ```

2. **Run the setup** If you want to customize the setup, run without `--recommended` and follow the prompts. Later, you also need to manually add 
(1) agents, (2) doc sync hook, and (3) save-tokens hooks if you want those features. Also, by default --recommended will prioritize Claude (unless only Codex Cli is installed). You will need to run `doc init --target=codex` to get the Codex agent and commands as well.

   ```bash
   npx aspens doc init --recommended
   ```

3. **Verify** freshness and coverage. This run a quick scan, checks if you docs are stale, or if you are missing useful funcitonality that aspens offers.

   ```bash
   npx aspens doc impact
   ```

4. **Keep it synced** on every commit. This is really the bread and butter of aspens — it keeps your docs up to date as your code evolves. You can run this manually, but we recommend installing the git hook so it runs automatically.

   ```bash
   npx aspens doc sync --install-hook
   ```

</Steps>

## Add agents and commands

```bash
npx aspens add agent all             # Install all 11 bundled AI agents
npx aspens customize agents          # Tailor agents with your project's context
npx aspens add command dev-docs      # Add slash commands
```
