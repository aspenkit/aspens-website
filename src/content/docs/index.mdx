## What <code>doc init</code> writes

<div class="file-manifest file-manifest-single">
	<div class="manifest-group">

		<ul class="manifest-list">

<section id="faq">

## FAQ

<div class="faq-list">
	<details class="faq-item">
		<summary>Does aspens require hosting or any third party services?</summary>
		<div class="faq-answer">No. Aspens is an open-source CLI. You use your own configured assistant for generation commands.</div>
	</details>
	<details class="faq-item">
		<summary>Which assistants does it support today?</summary>
		<div class="faq-answer">Claude Code and Codex are first-class targets. Aspens can generate target-specific repo context for both from a shared scan of the codebase.</div>
	</details>
	<details class="faq-item">
		<summary>What parts of Aspens use an LLM?</summary>
		<div class="faq-answer"><code>aspens scan</code> is deterministic. Commands that generate or refresh context, such as <code>doc init</code> and <code>doc sync</code>, use your configured coding-assistant backend.</div>
	</details>
	<details class="faq-item">
		<summary>Who is Aspens for?</summary>
		<div class="faq-answer">For anyone using AI coding tools regularly. The value shows up even for a solo maintainer if you want the repo context to persist across sessions.</div>
	</details>
	<details class="faq-item">
		<summary>Can I just try it without messing my setup or generating files?</summary>
		<div class="faq-answer">Yes. Run any generation command with <code>--dry-run</code> to preview what would be written without touching your repo.</div>
	</details>
	<details class="faq-item">
		<summary>Can it modify my source code?</summary>
		<div class="faq-answer">No. All writes are restricted to .claude/, .agents/, instruction files (CLAUDE.md/AGENTS.md), and .aspens.json. A path sanitizer enforces this — it cannot touch your application code.</div>
	</details>
	<details class="faq-item">
		<summary>Is the generated output committed to my repo?</summary>
		<div class="faq-answer">That's up to you. Skills and instruction files are regular files you can review, edit, or gitignore. Graph artifacts are auto-gitignored by default.</div>
	</details>
	<details class="faq-item">
		<summary>Is it open source?</summary>
		<div class="faq-answer">Yes. You can audit the full codebase on GitHub before running it.</div>
	</details>
</div>

document.querySelectorAll('.install-copy').forEach(function(btn) {

			if (copyIcon) copyIcon.style.display = 'none';
			if (checkIcon) checkIcon.style.display = '';
			setTimeout(function() {
				btn.classList.remove('copied');
				if (copyIcon) copyIcon.style.display = '';
				if (checkIcon) checkIcon.style.display = 'none';
			}, 2000);
		});
	});
});
